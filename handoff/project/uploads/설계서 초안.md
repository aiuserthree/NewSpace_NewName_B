
# 공간 이름 공모전 웹페이지 기획서 (초안)

> 본 문서는 회의록을 기반으로 작성된 **사용자 관점 · 비즈니스 로직 중심**의 기획 초안입니다.
> 개발 아키텍처/기술 구조는 제외하고, 화면 흐름과 정책 위주로 정리했습니다.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목적 | 신축·리모델링 공간의 이름을 교인 참여로 공모 |
| 공모 대상 공간 | ① 본당(신축, 조감도 제공) ② 소예배실(사진 제공) ③ 새가족부실 |
| 참여 방식 | 카카오톡 채널로 링크 발송 → 온라인 제출 |
| 성격 | 1차 온라인 공모 = 사전 조사 (이후 오프라인 2차 스티커 투표로 확정) |
| 링크 발송 예정 | 이번 주 토요일 |
| 디자인 방향 | 심플·깔끔, 이미지(사진/조감도)를 전면 배치해 시각 효과 강조 |

## 2. 핵심 정책 (비즈니스 로직)

### 2-1. 제출 규칙
- 1인당 필수 1개 + 추가 1개 = 최대 2개의 이름을 제출할 수 있다.
- 제출은 세 공간의 이름을 한 화면에서 한 번에 진행한다.
- 각 공간마다 설명 문구를 제공해 이름 제안의 방향을 안내한다.
  - 예) 소예배실: "청년·청소년을 위한 액티브한 공간"
- 예시 이름은 제공하지 않는다. (참여자 창의성 저해 방지)

### 2-2. 중복 방지 & 조회
- 핸드폰 번호를 고유 키(key)로 사용하여 중복 신청을 방지한다.
- 이미 제출한 번호로 재접근 시:
  - "이미 신청하신 분입니다" 안내
  - [내용 확인하기] 버튼 제공 → 본인이 제출한 내용 조회
- 수정 기능은 (현재 정책상) 구현하지 않는다. 제출 = 최종 완료.
  - ※ 조회 기능은 포함 확정, 수정 포함 여부는 추후 결정 (확장 여지 남김)

### 2-3. 데이터 & 후속 절차
- 제출 데이터는 엑셀로 추출 가능 (요청 시 담당자가 제공).
- 온라인 공모 종료 후 각 공간별 최다 제안 상위 3개 후보 선정.
- 선정 후보를 입구 대자보 게시 → 교인 스티커 투표로 최종 결정 (크로스 체크).

## 3. 정보 구조 (IA)

    공간 이름 공모전 (Single Page + 2 Step Flow)
    │
    ├── [0] 랜딩 / 인트로 영역
    │     └─ 공모전 소개, 참여 방법, 시작하기 버튼
    │
    ├── [STEP 1] 신청자 정보 입력
    │     ├─ 이름
    │     └─ 연락처(핸드폰 번호)  ← 중복 확인 키
    │         ├─ (신규) → STEP 2 진행
    │         └─ (기존) → "이미 신청하신 분입니다" + [내용 확인하기]
    │
    ├── [STEP 2] 이름 아이디어 제출 (스크롤형 단일 페이지)
    │     ├─ ① 본당 (조감도 + 설명)     : 이름 1개(필수) + 1개(선택)
    │     ├─ ② 소예배실 (사진 + 설명)   : 이름 1개(필수) + 1개(선택)
    │     └─ ③ 새가족부실 (사진 + 설명) : 이름 1개(필수) + 1개(선택)
    │         └─ [제출하기]
    │
    ├── [완료] 제출 완료 화면
    │     └─ "완료되었습니다" + 다음 일정(2차 투표) 안내
    │
    └── [조회] 내 제출 내용 확인 화면
          └─ 핸드폰 번호로 조회한 제출 내역 표시 (읽기 전용)

> 흐름 원칙: '다음' 버튼을 반복적으로 누르는 다중 페이지 방식 대신,
> 스크롤 기반 단일 페이지 + 최소 단계(정보입력 → 이름제출)로 이탈을 방지한다.

## 4. 메뉴 / 화면 구조

| 순서 | 화면 | 주요 구성 요소 | 사용자 액션 |
|------|------|----------------|-------------|
| 0 | 인트로 | 공모 취지 · 참여 방법 · 대상 공간 요약 | [참여 시작] |
| 1 | 신청자 정보 | 이름, 핸드폰 번호 입력 | [다음] / 중복 시 [내용 확인하기] |
| 2 | 이름 제출 | 공간별 이미지·설명, 이름 입력(필수1+선택1) | [제출하기] |
| 3 | 제출 완료 | 완료 메시지, 2차 투표 일정 안내 | (종료) |
| 4 | 내용 조회 | 제출 내역 읽기 전용 표시 | (닫기) |

화면 배치 순서(스크롤): 본당(조감도) → 소예배실(사진) → 새가족부실(사진)
- 미완공 상태인 본당을 최상단에 배치하고 조감도로 상상 유도.

---

## 5. 상세 사용자 흐름 (User Flow)

### 5-1. 신규 참여자
    링크 클릭 → 인트로 확인 → [참여 시작]
       → 이름·번호 입력 → (신규 확인) → [다음]
       → 세 공간 이름 입력(각 필수1, 선택1) → [제출하기]
       → 제출 완료 화면(완료 메시지 + 2차 투표 안내)

### 5-2. 기존 참여자 (동일 번호 재접근)
    링크 클릭 → 인트로 → [참여 시작]
       → 번호 입력 → (중복 감지)
       → "이미 신청하신 분입니다" + [내용 확인하기]
       → 내 제출 내용 조회(읽기 전용)
